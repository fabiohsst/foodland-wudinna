Revenue & GP% by product — last 30 days
Ranked list of all items by revenue in the current trading period.
GROUP BYJOIN
SELECT
    p.name,
    p.sub_dept,
    ROUND(SUM(s.sales_ex_gst), 2)  AS revenue,
    ROUND(SUM(s.quantity),     1)  AS units_sold,
    ROUND(AVG(s.gp_pct) * 100, 1)  AS avg_gp_pct
FROM   fact_sales  s
JOIN   dim_product p ON s.product_id = p.product_id
WHERE  s.date_id >= date('now', '-30 days')
GROUP BY p.product_id
ORDER BY revenue DESC
LIMIT  20;

Tip: ref_item_price holds the current sell/cost price. fact_invoice is the full history of every price change from an invoice.
All current prices with actual GP% from recent sales
Compare the price file GP% against what the POS actually recorded.
JOINsubquery
SELECT
    p.name,
    r.sell_price,
    r.cost_price,
    ROUND((r.sell_price - r.cost_price) / r.sell_price * 100, 1) AS target_gp_pct,
    ROUND(AVG(s.gp_pct) * 100, 1)                                 AS actual_gp_pct
FROM   ref_item_price r
JOIN   dim_product    p ON r.product_id  = p.product_id
LEFT  JOIN fact_sales s ON s.product_id  = p.product_id
                       AND s.date_id >= date('now', '-60 days')
WHERE  r.sell_price IS NOT NULL
GROUP BY p.product_id
ORDER BY ABS(target_gp_pct - actual_gp_pct) DESC
LIMIT  30;

Items with GP% below 35% in the last 4 weeks
Spot items that may need a price review — the 40% target is our floor.
HAVINGfilter
SELECT
    p.name,
    p.sub_dept,
    ROUND(AVG(s.gp_pct) * 100, 1) AS avg_gp_pct,
    ROUND(SUM(s.sales_ex_gst), 2)  AS revenue
FROM   fact_sales  s
JOIN   dim_product p ON s.product_id = p.product_id
WHERE  s.date_id >= date('now', '-28 days')
GROUP BY p.product_id
HAVING avg_gp_pct < 35
ORDER BY avg_gp_pct ASC;
Weekly revenue — last 12 weeks
See the revenue trend week by week to spot seasonal patterns.
strftimeGROUP BY
SELECT
    strftime('%Y-W%W', s.date_id)  AS week,
    ROUND(SUM(s.sales_ex_gst), 2) AS revenue,
    ROUND(SUM(s.gp_dollars),   2) AS gross_profit,
    COUNT(DISTINCT s.date_id)       AS trading_days
FROM   fact_sales s
WHERE  s.date_id >= date('now', '-84 days')
GROUP BY week
ORDER BY week;
Same product: this year vs last year (by month)
Year-on-year comparison using CASE WHEN to pivot the years into columns.

SELECT
    strftime('%m', s.date_id) AS month,
    ROUND(SUM(CASE WHEN strftime('%Y', s.date_id) = '2025'
                  THEN s.quantity ELSE 0 END), 1) AS qty_2025,
    ROUND(SUM(CASE WHEN strftime('%Y', s.date_id) = '2026'
                  THEN s.quantity ELSE 0 END), 1) AS qty_2026
FROM   fact_sales  s
JOIN   dim_product p ON s.product_id = p.product_id
WHERE  p.name = 'BANANAS PER KG'
GROUP BY month
ORDER BY month;
Top 10 products by revenue growth (2025 → 2026 YTD)
Which items are growing fastest? Uses a CTE to compute both years cleanly.
CTEWITH
WITH by_year AS (
    SELECT
        p.product_id,
        p.name,
        strftime('%Y', s.date_id)    AS yr,
        SUM(s.sales_ex_gst)           AS revenue
    FROM fact_sales  s
    JOIN dim_product p ON s.product_id = p.product_id
    WHERE strftime('%m-%d', s.date_id) <= strftime('%m-%d', 'now')
    GROUP BY p.product_id, yr
)
SELECT
    a.name,
    ROUND(a.revenue, 2)                              AS rev_2025,
    ROUND(b.revenue, 2)                              AS rev_2026,
    ROUND((b.revenue - a.revenue) / a.revenue * 100) AS growth_pct
FROM   by_year a
JOIN   by_year b ON a.product_id = b.product_id
                AND a.yr = '2025' AND b.yr = '2026'
WHERE  a.revenue > 100
ORDER BY growth_pct DESC
LIMIT  10;
dim_date is your friend here. Because it has every calendar day, a LEFT JOIN from dim_date onto fact_sales will show days with no sales as NULLs — that's how you find gaps.
Find trading days with missing data
Shows every day the store should have been open but has no sales records.
LEFT JOINIS NULL
SELECT
    d.date_id,
    d.day_name,
    d.holiday_name
FROM   dim_date d
LEFT  JOIN (
    SELECT DISTINCT date_id FROM fact_sales
) s ON d.date_id = s.date_id
WHERE  d.is_trading = 1
  AND  d.date_id >= '2026-01-01'
  AND  d.date_id <= date('now')
  AND  s.date_id IS NULL
ORDER BY d.date_id;
Products not sold in the last 14 days
Items that have gone quiet — potential stockout or waste write-off.
LEFT JOINIS NULL
SELECT
    p.name,
    p.sub_dept,
    MAX(s_all.date_id) AS last_sold
FROM   dim_product p
JOIN   fact_sales  s_all ON s_all.product_id = p.product_id
LEFT  JOIN fact_sales s_rec ON s_rec.product_id = p.product_id
                          AND s_rec.date_id >= date('now', '-14 days')
WHERE  s_rec.sale_id IS NULL
GROUP BY p.product_id
ORDER BY last_sold DESC;
Current stock vs average daily demand
Days of cover per product — cross-references stock snapshot with recent sales rate.
CTEJOIN
WITH avg_demand AS (
    SELECT
        product_id,
        AVG(quantity) AS avg_daily_qty
    FROM fact_sales
    WHERE date_id >= date('now', '-21 days')
    GROUP BY product_id
)
SELECT
    p.name,
    ROUND(fs.stock,          1)  AS stock_on_hand,
    ROUND(a.avg_daily_qty,   2)  AS avg_daily_qty,
    ROUND(fs.stock / NULLIF(a.avg_daily_qty, 0), 1) AS days_cover
FROM   fact_stock   fs
JOIN   dim_product  p  ON fs.product_id = p.product_id
JOIN   avg_demand   a  ON fs.product_id = a.product_id
WHERE  fs.date_id = (SELECT MAX(date_id) FROM fact_stock)
ORDER BY days_cover ASC;

Quick rule: Always join through product_id — every table links back to dim_product. For date filtering, use date_id (text, ISO format YYYY-MM-DD) directly in WHERE clauses.
